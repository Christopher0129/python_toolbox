# Outbox / Inbox 模式专题

这个专题关注数据库写入和消息发布之间的一致性问题，避免“业务成功了但消息没发出去”或“消息重复消费”的尴尬状态。

## 核心模式

| 模式 | 说明 |
| --- | --- |
| Outbox | 在本地事务里先写业务数据和待发送事件 |
| Inbox | 消费侧记录已处理消息，防止重复执行 |

## 适用场景

- 订单创建后要发布领域事件
- Webhook 事件需要可靠转发
- 多服务之间基于事件保持最终一致

## 实战建议

- Outbox 表需要清晰的发送状态和重试字段
- Inbox 去重键要稳定且可查询
- 事件投递失败要有重试与死信策略
