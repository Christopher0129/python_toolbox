# 测试矩阵专题

这个专题关注测试不只在一个 Python 版本、一个依赖集合、一个数据库上通过，而是如何系统覆盖关键组合。

## 适用场景

- 需要兼容多个 Python 版本的库项目
- 依赖版本差异明显的服务端项目
- 多数据库、多后端、多环境支持的组件

## 常见主题

- Python 版本矩阵
- 依赖上界 / 下界测试
- 数据库与中间件后端矩阵
- smoke test 与 full test 分层
- CI 并行和缓存优化

## 设计要点

- 不要盲目穷举所有组合，优先覆盖风险最高的边界
- 上界和下界都测，才能更早暴露兼容漂移
- 矩阵规模要和运行成本平衡，但关键组合不能省
- 失败结果要能快速映射到版本差异或环境差异
