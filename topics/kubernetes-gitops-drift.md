# Kubernetes 配置漂移专题

这个专题关注 GitOps 或声明式集群里，线上实际状态与期望配置逐渐偏离时的发现与治理。

## 适用场景

- 线上紧急手改过资源，之后没有回写 Git
- 同一资源被多个控制器或脚本反复覆盖
- 发布后集群状态和仓库配置长期不一致

## 常见主题

- 漂移检测
- 手工改动
- 多控制器冲突
- 回写策略
- 差异审计

## 设计要点

- 漂移不只是“内容不一样”，还包括谁改的、为什么改、是否应该保留。
- 对线上紧急修复要有回写或过期处理机制，否则会积累隐性状态。
- 漂移检测要区分无害差异和真正影响治理的差异。
- GitOps 成功的前提是变更入口收敛，而不是只部署一个同步工具。
