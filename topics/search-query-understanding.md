# 搜索 Query Understanding 专题

这个专题关注在真正检索前，对用户查询做意图、实体、约束和改写理解。

## 适用场景

- 用户查询短、模糊、口语化
- 同一关键词在不同领域有不同含义
- 需要把自然语言转成结构化过滤和排序偏好

## 常见主题

- 查询归一化
- 实体识别
- 意图分类
- 约束抽取
- 改写与扩展

## 设计要点

- Query Understanding 的目标不是“语言学上更正确”，而是让后续检索更稳定。
- 识别出的结构化约束要能追溯和审计，避免黑盒改写污染结果。
- 查询理解出错时，系统应能降级回原查询，而不是强行套错模板。
- 短查询、高频头部查询和长尾自然语言查询的策略通常不同。
