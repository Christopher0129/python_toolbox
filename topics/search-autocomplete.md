# 搜索自动补全专题

这个专题关注用户输入尚未完成时，如何提供高相关、低延迟、可控噪声的查询建议。

## 适用场景

- 搜索框需要边输入边提示
- 文档标题、商品名或命令词存在明显前缀模式
- 用户拼写不稳定，常依赖补全减少输入成本

## 常见主题

- 前缀匹配
- `suggest` / completion
- 热门查询
- 拼写纠错
- 新鲜度与去噪

## 设计要点

- 自动补全追求的是输入辅助，不是完整搜索结果替代。
- 建议词要平衡热门度、可解释性和内容新鲜度，避免旧热点长期垄断。
- 补全索引通常需要单独设计字段和更新链路，不能完全复用正文检索字段。
- 对高频噪声词、敏感词和低质量 query log 要有清洗策略。
