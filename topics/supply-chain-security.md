# 供应链安全专题

这个专题关注依赖、镜像、构建产物和发布流程本身的安全，而不只是运行时权限。

## 适用场景

- Python 服务依赖众多、升级频繁的工程团队
- 容器镜像构建和多环境交付
- 需要审计依赖来源、漏洞和制品完整性的生产环境

## 常见主题

- 依赖锁定与来源控制
- 漏洞扫描与升级策略
- SBOM 与制品清单
- 镜像签名与制品校验
- CI/CD 凭证最小权限

## 设计要点

- 把“可复现构建”当成安全能力，而不是只看功能成功
- 依赖升级要有灰度和回滚，不是一次性全量刷新
- 私有仓库、镜像仓库和密钥系统的权限边界要分开
- 安全基线最好进入 CI，而不是靠人工巡检
