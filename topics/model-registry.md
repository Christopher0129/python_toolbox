# 模型注册专题

这个专题关注模型不只是训练完就结束，还要有可追溯的版本、别名、审核状态和回滚路径。

## 常见能力

| 能力 | 说明 |
| --- | --- |
| 模型版本 | 每次可部署产物都能独立编号 |
| 标签 / 别名 | 标识 `candidate`、`prod` 等状态 |
| 产物关联 | 关联训练 run、参数、数据版本 |
| 审核流程 | 部署前记录验证结果和责任人 |

## 常见流程

1. 在实验跟踪中记录训练过程
2. 产出模型文件和元数据
3. 注册到模型仓库
4. 完成验证、打标签、提升环境
5. 出问题时按版本回滚
