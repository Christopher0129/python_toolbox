# 数据血缘专题

这个专题关注数据从哪里来、被谁加工、流向哪里，以及变化如何影响下游。

## 适用场景

- 表很多，变更影响面难评估
- 需要做数据治理和事故追踪
- 指标口径在多个作业中传递

## 常见主题

- 表级血缘
- 列级血缘
- SQL 解析
- 作业依赖图
- 变更影响分析

## 设计要点

- 血缘不是为了“画图好看”，而是服务变更治理和事故定位
- 自动血缘提取要和命名规范、SQL 质量一起建设
- 动态 SQL 和脚本拼接会显著提高血缘难度
- 列级血缘很有价值，但维护成本也更高
