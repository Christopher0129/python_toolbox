# 发布策略专题

这个专题关注服务如何从旧版本切到新版本，以及出问题时如何尽快收敛影响范围。

## 常见策略

| 策略 | 说明 |
| --- | --- |
| 直接替换 | 最简单，但风险集中 |
| 蓝绿发布 | 新旧环境并存，切流更清晰 |
| 金丝雀发布 | 小流量先验证，再逐步放量 |
| 滚动发布 | 分批替换实例，常见于容器环境 |

## 决策要点

- 是否能快速回滚
- 是否有完善监控和告警
- 是否能按用户、区域或实例逐步放量
- 数据库变更是否与应用版本兼容
