# 缓存稳定性专题

这个专题关注缓存击穿、雪崩、穿透、热点失衡等问题，以及如何在 Python 服务中落地保护策略。

## 适用场景

- 高并发热点查询
- Redis 作为主缓存层的服务
- 大促、突发流量和定时失效场景

## 常见主题

- 缓存击穿与互斥重建
- 缓存雪崩与失效分散
- 缓存穿透与空值缓存
- 热 key 识别与降级
- 预热、刷新和双写一致性

## 设计要点

- 不要把缓存命中率当唯一指标，关键是后端承压是否可控
- 热 key 保护和降级路径要提前设计
- TTL 策略要结合业务时效和重建成本
- 缓存问题通常和限流、重试、熔断一起看
