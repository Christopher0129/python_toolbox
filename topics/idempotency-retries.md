# 幂等与重试专题

这个专题关注失败后的恢复策略，但核心不是“无限重试”，而是先保证操作可重复执行且结果可控。

## 关键概念

| 概念 | 作用 |
| --- | --- |
| 幂等 | 相同请求重复执行，结果不被重复放大 |
| 重试 | 对暂时性失败再次尝试 |
| 退避 | 控制重试间隔，避免放大故障 |
| 去重 | 防止消息或回调被重复消费 |

## 常见场景

- 支付和下单接口
- Webhook 回调处理
- 消息队列消费
- 外部 API 调用失败恢复

## 实战建议

- 写请求优先设计幂等键，再做自动重试
- 只对超时、连接失败、限流等暂时性错误重试
- 指数退避最好带抖动，避免同一时刻集体重试
- 重试上限、超时和告警阈值要一起设计
