# API Key 治理专题

这个专题关注面向用户、租户或服务发放的 API Key 如何创建、限制、轮换和审计。

## 适用场景

- 对外开放 API，需要为调用方分配独立凭证
- 不同客户需要不同配额、权限和失效策略
- 密钥泄露后的吊销与轮换流程不清晰

## 常见主题

- key 发放
- 前缀与哈希存储
- 权限范围
- 配额与限流
- 吊销与轮换

## 设计要点

- API Key 应只展示一次明文，服务端持久化时优先保存哈希或摘要。
- key 的身份、权限和配额应与租户、应用和环境显式绑定。
- 凭证治理必须和审计日志、告警策略、速率限制一起设计。
- 面向内部服务的长期 API Key 应尽量被更强身份方案替代。
