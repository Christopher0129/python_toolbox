# 断路器与隔离专题

这个专题关注一个依赖出问题时，如何避免把整个服务拖死。

## 核心模式

| 模式 | 作用 |
| --- | --- |
| 断路器 | 失败过多时短路下游调用 |
| 超时 | 防止线程、协程被慢依赖长期占住 |
| 隔离舱壁 | 把不同依赖或任务的资源分开 |
| 降级 | 下游失败时返回受控替代结果 |

## 常见场景

- 调用慢数据库或外部 HTTP 服务
- 第三方支付、短信、推荐服务不稳定
- 某一类后台任务堆满工作线程
