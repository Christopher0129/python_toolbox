# 事务与隔离级别专题

这个专题关注数据库事务到底保证什么、隔离级别怎么影响并发读写，以及 Python 服务如何避免把问题留给线上。

## 适用场景

- 下单、扣库存、记账等强一致业务
- 多 worker 并发更新同一批记录
- 查询结果受并发写入影响

## 常见主题

- 原子性与回滚
- 脏读、不可重复读、幻读
- 悲观锁与乐观锁
- 长事务与锁等待
- 重试与幂等

## 设计要点

- 先定义业务一致性要求，再选择隔离级别和锁策略
- 应用层重试必须和事务语义匹配，不能盲目重放写请求
- 长事务会放大锁冲突、回滚成本和资源占用
- 事务失败后要保留足够上下文，便于定位锁竞争与死锁
