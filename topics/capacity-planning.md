# 容量规划专题

这个专题关注在流量增长前估算系统能承受多少，并提前安排扩容、限流和成本边界。

## 适用场景

- 大促、活动、批处理高峰
- 新功能上线前容量评估
- 模型服务或数据库成本增长明显

## 常见主题

- QPS / TPS 预算
- 连接池和 worker 数量
- CPU / 内存 / IO 瓶颈
- 扩容阈值
- 成本与性能平衡

## 设计要点

- 容量规划要从真实业务路径出发，而不是只看单机 benchmark
- 上游流量、应用层并发和下游容量必须一起估算
- 扩容策略要同时覆盖资源、配置和依赖配额
- 规划结果要体现在告警阈值和限流默认值中
