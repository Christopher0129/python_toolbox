# 开发环境热重载专题

这个专题关注本地开发时如何在代码变化后自动重载服务、脚本或 worker，提升反馈速度。

## 适用场景

- Web 服务本地联调
- CLI / worker / 数据任务开发
- 多进程或多目录项目的开发态反馈优化

## 常见主题

- 文件监听与自动重启
- 代码生成与模板刷新
- 前后端或多服务联动开发
- 排除目录和噪声事件控制
- 开发态与生产态运行差异

## 设计要点

- 热重载要明确只服务开发环境，不应混入生产运行逻辑
- 自动重启要控制监听范围，否则会被日志、缓存文件干扰
- 多服务联调时要统一启动脚本和端口分配
- 文件监听只是入口，配置注入和依赖初始化也要够快
