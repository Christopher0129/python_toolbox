# 服务网格专题

这个专题关注服务间流量治理从应用代码外移到基础设施侧之后，Python 服务需要理解的接入与协作边界。

## 适用场景

- 微服务规模较大
- 统一做流量治理、mTLS 和观测
- 发布策略依赖流量切分和金丝雀能力

## 常见主题

- sidecar 与代理链路
- mTLS 和身份认证
- 流量镜像与金丝雀
- 重试、超时和熔断下沉
- 指标与追踪透传

## 设计要点

- 引入服务网格并不等于应用侧不需要治理
- 应用超时、重试和网格层策略要避免叠加放大
- Header 透传、身份信息和 tracing context 需要统一约定
- 网格更适合治理共性流量问题，不替代领域逻辑
