# 限流专题

这个专题关注服务在面对高并发、误用和恶意调用时，如何通过配额控制保护自己。

## 常见维度

| 维度 | 说明 |
| --- | --- |
| 按 IP | 简单粗暴，适合匿名接口 |
| 按用户 / Token | 更适合登录后接口 |
| 按接口 | 给高成本路由单独配额 |
| 按共享资源 | 保护数据库、下游服务、批处理资源 |

## 常见策略

- 固定窗口
- 滑动窗口
- 令牌桶 / 漏桶
- 全局限流和路由级限流组合

## 实战建议

- 超限响应统一返回 `429`
- 限流键优先选择业务身份，而不是只看来源 IP
- 重要限流规则要配合监控，不然容易误伤正常流量
- 对管理接口、批处理回调和内部调用要区分配额
