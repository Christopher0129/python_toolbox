# RAG 评测专题

这个专题关注 RAG 系统不只测生成结果，还要分开评估检索、重排、上下文选择和最终回答。

## 适用场景

- 检索方案、切块方案或 prompt 频繁调整
- 需要比较不同向量库、重排模型或生成模型
- 上线后想监控质量回退

## 常见主题

- 检索命中率
- 上下文相关性
- 答案正确性
- 引用一致性
- 离线评测集

## 设计要点

- 评测数据集要覆盖真实问题分布和失败样本
- 先定位是检索问题、排序问题还是生成问题
- 指标要能对比版本，而不是只看单次绝对分
- 线上反馈和离线评测要形成闭环
